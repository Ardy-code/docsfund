title: Software Requirement


## Functional Requirements (Kebutuhan Fungsional)

Functional requirements adalah fitur atau fungsi spesifik yang harus ada dalam aplikasi agar dapat memenuhi kebutuhan pengguna. Contoh kebutuhan fungsional untuk aplikasi pencatatan keuangan:

**Autentikasi Pengguna**
- Pengguna harus mendaftar akun menggunakan email dan password
- Pengguna dapat masuk (login) untuk mengakses data keuangan pribadi mereka.

**Operasi CRUD**
- Pengguna dapat membuat catatan pemasukan dan pengeluaran
- Pengguna dapat mengedit atau menghapus catatan keuangan yang sudah dibuat
- Pengguna dapat melihat semua catatan yang telah dibuat dalam bentuk tabel

**Manajemen Anggaran (Wallet)
- Pengguna dapat membuat anggaran sesuai dengan uang yang ingin di atur 
- Sistem dapat menampilkan sisa anggaran berdasarkan total pengeluaran

## Non-Functional Requirement (Kebutuhan Non-Fungsional)

Non-functional requirements adalah aspek teknis yang mendukung performa, keamanan, dan skalabilitas aplikasi. Contohnya:

**Keamanan**
- Sistem harus menggunakan enkripsi (misalnya, hashing untuk penyimpanan password)
- Tidak ada data pengguna yang dapat diakses tanpa autentikasi yang sah

**Skalabilitas**
- Aplikasi harus dirancang agar mudah diperluas untuk mendukung fitur tambahan, seperti budgeting berbasis AI di masa depan.

**Kompatibilitas**
- Aplikasi harus dapat berjalan di browser modern seperti Chrome, Firefox, dan Safari
- Aplikasi harus responsif dan dapat digunakan pada perangkat desktop maupun ponsel
